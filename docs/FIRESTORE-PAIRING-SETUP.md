# Firestore Setup for Pairing & Relationships System

This document explains the Firestore collections, indexes, and security rules for the Pairing & Relationships system.

## Collections

### inviteCodes

Stores time-limited invite codes generated by parents.

**Document Structure:**

```javascript
{
  code: string,              // 8-character uppercase alphanumeric
  parentUid: string,         // Firebase Auth UID of parent
  createdAt: Timestamp,      // Firestore server timestamp
  expiresAt: Timestamp,      // 24 hours after createdAt
  usedCount: number          // Number of times redeemed (for analytics)
}
```

**Security Rules:**

- Read: Only the parent who created the code
- Create: Only authenticated parents for themselves
- Update/Delete: Not allowed (handled by Cloud Functions)

### relationships

Stores bidirectional relationships between parents and caregivers.

**Document Structure:**

```javascript
{
  parentUid: string,         // Firebase Auth UID of parent
  caregiverUid: string,      // Firebase Auth UID of caregiver
  createdAt: Timestamp,      // Firestore server timestamp
  createdBy: string          // UID of user who created (usually caregiver)
}
```

**Security Rules:**

- Read: Only users who are part of the relationship (parent or caregiver)
- Create: Not allowed (must use Cloud Function)
- Delete: Only users who are part of the relationship
- Update: Not allowed

## Composite Indexes

The following composite indexes are required for efficient queries:

1. **inviteCodes**: `code` (ASC) + `expiresAt` (DESC)

   - Used for: Validating invite codes during redemption

2. **inviteCodes**: `parentUid` (ASC) + `expiresAt` (DESC)

   - Used for: Finding active codes for a parent

3. **relationships**: `parentUid` (ASC) + `caregiverUid` (ASC)

   - Used for: Duplicate detection during relationship creation

4. **relationships**: `caregiverUid` (ASC)

   - Used for: Querying relationships for a caregiver

5. **relationships**: `parentUid` (ASC)
   - Used for: Querying relationships for a parent

## Deployment

### Deploy Security Rules

```bash
firebase deploy --only firestore:rules
```

### Deploy Indexes

```bash
firebase deploy --only firestore:indexes
```

### Deploy Both

```bash
firebase deploy --only firestore
```

## Testing Security Rules

You can test the security rules locally using the Firebase Emulator Suite:

```bash
# Start the emulator
firebase emulators:start

# Run tests against the emulator
npm test -- firestore.rules.test.js
```

## Notes

- The collections will be created automatically when the first document is written
- Indexes may take a few minutes to build after deployment
- Security rules are enforced immediately after deployment
- The `inviteCodes` collection should have a TTL policy or scheduled cleanup function to remove expired codes
